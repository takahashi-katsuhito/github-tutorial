# Git 概要

## 🧩 Gitとは？
**Git** は、ソースコードなどの変更履歴を管理するための **分散型バージョン管理システム（DVCS）**。  
複数人での開発や、過去の状態へのロールバックを簡単にするために使われる。

---

## 🔍 Gitの特徴

### 1. 分散型バージョン管理
- 開発者全員が **リポジトリの完全なコピー（履歴含む）** を保持する  
- オフラインでも履歴閲覧やコミットが可能  
- サーバが壊れても、別のクローンから復旧できる

### 2. スナップショット管理
- ファイルの差分ではなく、**プロジェクト全体のスナップショット**を保存  
- 変更がないファイルは再利用するため高速で効率的

### 3. 速い処理
- コミット・ブランチ作成・マージなどがローカルで完結するため高速

---

## 🌱 基本概念

### リポジトリ（Repository）
変更履歴（コミット）を保持するデータベース  
- **ローカルリポジトリ**：自分の PC 内  
- **リモートリポジトリ**：GitHub などのサーバ上

### コミット（Commit）
変更を記録する単位。  
メッセージとともにファイルの差分やスナップショットを保存。

### ブランチ（Branch）
作業の流れを分岐させる機能。  
例：`main`（安定版）と `feature/xxx`（新機能開発）

### マージ（Merge）
分岐したブランチを統合する操作。

### ステージ（Index / Staging Area）
コミットする前に変更を「一時的に溜める場所」。

---

## 🏃 基本的な Git コマンド

```bash
# 初期化
git init

# コードをリポジトリに追加
git add <file>

# 変更を記録
git commit -m "メッセージ"

# リモートリポジトリを追加
git remote add origin <URL>

# リモートへプッシュ
git push origin main

# ブランチ作成
git branch feature/my-feature

# ブランチ切り替え
git checkout feature/my-feature

# マージ
git merge feature/my-feature
